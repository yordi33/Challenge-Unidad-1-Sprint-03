¿De qué va este reto?

Trabajáis con un dataset de películas clásico (MovieLens) y construís un catálogo enriquecido en cuatro fases progresivas: análisis de datos con Pandas, consumo de una API REST (TMDB), traducción automática con Gemini y, si queréis ir más lejos, funcionalidades opcionales.

📁 Datasets de partida

  • movies.csv — títulos y géneros
  • ratings.csv — valoraciones de usuarios
  • tags.csv — etiquetas libres
  • links.csv — IDs externos (TMDB, IMDb)

Ruta: Team_Challenges/TC_01_Sprint_03_04/data/

🔑 Credenciales necesarias

  • TMDB: TMDB_API_KEY o TMDB_READ_ACCESS_TOKEN
  • Gemini: GEMINI_API_KEY
Nunca subáis claves al repositorio. Usad .env + .gitignore o getpass.

Organización del equipo

Antes de escribir una sola línea de código, dedicad el primer día a establecer las bases de trabajo. La organización es la mitad del proyecto.

Reparto de tareas sugerido

Cada miembro lidera una parte, pero todos revisan el trabajo de los demás vía Pull Request.

👤 Persona A

Crea el repositorio, configura .gitignore, escribe el README inicial y lidera la Parte 1 (Pandas).

👤 Persona B

Configura las variables de entorno y lidera la Parte 2 (TMDB API).

👤 Persona C

Lidera la Parte 3 (Gemini). Si el equipo es de dos personas, repartid partes 2 y 3 entre ambas.

Las personas que no lideran una parte son las revisoras de esa PR. Así todos tocan todo el código.

🟢 main

Rama de producción. Solo contiene versiones estables y revisadas. Solo se actualiza mediante PR desde develop.

  • Protegida: requiere aprobación para merge
  • Nunca se hace git push directo aquí

🔵 develop

Rama de integración continua. Es vuestra rama de trabajo diario compartida.

  • Se crean siempre desde develop

Manejo de conflictos

Un conflicto ocurre cuando dos ramas modifican las mismas líneas. Resolución:

# 1. Actualizar tu rama con develop
git checkout feature/mi-rama
git pull origin develop

# 2. Si hay conflictos, Git marcará los ficheros
# Abre el fichero → edita las marcas:
#   <<<<<<< HEAD   ← tu código
#   =======
#   código de develop
#   >>>>>>> develop

# 3. Guardar, añadir y hacer commit
git add fichero_resuelto.py
git commit -m "fix: resolver conflictos con develop"
git push origin feature/mi-rama
Para evitar conflictos, haced git pull origin develop en vuestra rama frecuentemente. Cuanto más sincronizados estéis, menos conflictos tendréis.

Petición HTTP a la API de TMDB

Enriquecer 10 películas con datos de The Movie Database

Endpoint a usar

GET https://api.themoviedb.org/3/movie/{tmdb_id}

El tmdb_id viene de la columna tmdbId del fichero links.csv.

Tareas

  • Construir movies10: las 10 primeras filas de movies que tengan tmdbId (merge con links)
  • Implementar fetch_movie_details(tmdb_id) que devuelva al menos overview y homepage (cadena vacía si vienen nulos)
  • Recorrer movies10 y añadir ambas columnas al DataFrame

Autenticación

Opción A — API Key

params = {"api_key": os.getenv("TMDB_API_KEY")}
resp = requests.get(url, params=params)

Opción B — Bearer Token

headers = {
    "Authorization": f"Bearer {os.getenv('TMDB_READ_ACCESS_TOKEN')}"
}
resp = requests.get(url, headers=headers)
Probad el endpoint en Postman antes de codificar. Es mucho más rápido depurar allí que en el notebook.
No guardéis claves en el notebook ni en ningún fichero que se suba al repositorio.
03

Sinopsis en español con Gemini

Traducir y resumir los overview de TMDB usando la API de Gemini

Requiere haber completado la Parte 2 (movies10 con columna overview) y pip install google-genai.

Tareas

  • Configurar GEMINI_API_KEY (variable de entorno, getpass o .env)
  • Crear el cliente google-genai
  • Implementar summarize_overview_es(overview, title="") que devuelva un resumen en español de máximo 2 frases
  • Si overview está vacío, devolver cadena vacía sin llamar a la API
  • Recorrer movies10 y añadir columna overview_es
  • Mostrar title, overview (recorte) y overview_es para 3 películas

Esqueleto de la función

from google import genai
import os

client = genai.Client(api_key=os.getenv("GEMINI_API_KEY"))
MODEL  = "gemini-2.0-flash"

def summarize_overview_es(overview: str, title: str = "") -> str:
    if not overview.strip():
        return ""
    prompt = (
        f"Película: {title}\n"
        f"Sinopsis original: {overview}\n\n"
        "Resume esta sinopsis en español en máximo 2 frases."
    )
    resp = client.models.generate_content(model=MODEL, contents=prompt)
    return resp.text.strip()
